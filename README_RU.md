# Развертывание Сервисов VPS

Развертывание прокси-сервисов одной командой на VPS с низкими ресурсами (1CPU, 1GB RAM, 5GB Storage).

## Сервисы

- **HTTP Прокси (3proxy)** - HTTP прокси с аутентификацией
- **Telegram Прокси (telemt)** - MTProto прокси с маскировкой FakeTLS и сайтом-прикрытием
- **AmneziaWG 2.0** - WireGuard VPN (подготовлено для будущей реализации)

## Быстрый Старт

### Развертывание в Одну Команду

```bash
curl -fsSL https://raw.githubusercontent.com/unton83/VPS_VPN_PROXY/master/deploy.sh | bash
```

### Ручное Развертывание

```bash
# Клонировать репозиторий
git clone https://github.com/unton83/VPS_VPN_PROXY.git
cd VPS_VPN_PROXY

# Сделать скрипт развертывания исполняемым
chmod +x deploy.sh

# Запустить развертывание
./deploy.sh
```

## Требования

- VPS с Ubuntu 20.04+ / Debian 11+
- Docker + Docker Compose v2
- Для Telegram прокси: домен с A-записью, указывающей на IP VPS
- Открытые порты: 80, 443 (Telegram), 8080 (HTTP Прокси)

## Конфигурация Сервисов

### HTTP Прокси (3proxy)
- **HTTP**: `VPS_IP:8080`
- **Аутентификация**: Автоматически сгенерированные логины и пароли
- **Пользователи**: user1, user2, user3 (учетные данные показаны после развертывания)

### Telegram Прокси (telemt)
- **Порт**: 443 (HTTPS)
- **Протокол**: MTProto с маскировкой FakeTLS
- **Сайт-прикрытие**: Обслуживается на порту 80
- **SSL**: Сертификаты Let's Encrypt с автообновлением
- **Прокси-ссылки**: Генерируются автоматически после развертывания

## Варианты Развертывания

Скрипт развертывания предлагает следующие варианты:

1. **Только HTTP Прокси** - Развернуть только сервис 3proxy
2. **Только Telegram Прокси** - Развернуть только сервис telemt  
3. **Оба сервиса** - Развернуть оба прокси (HTTP и Telegram)
4. **Выход** - Отменить развертывание

## Использование Ресурсов

Все сервисы оптимизированы для VPS с низкими ресурсами:

| Сервис | Лимит CPU | Лимит Памяти | Сеть |
|--------|-----------|--------------|------|
| 3proxy | 0.3 ядра | 256MB | proxy-network |
| Nginx (Telegram) | 0.25 ядра | 256MB | telegram-network |
| Telemt | 0.4 ядра | 256MB | telegram-network |
| Certbot | 0.2 ядра | 128MB | telegram-network |

**Всего**: ~1.15 ядер, ~896MB RAM (в пределах 1CPU/1GB)

## Команды Управления

```bash
# Проверить статус сервисов
docker compose ps

# Просмотреть логи
docker compose logs -f 3proxy      # Логи HTTP прокси
docker compose logs -f telemt      # Логи Telegram прокси
docker compose logs -f web         # Логи Nginx

# Перезапустить сервисы
docker compose restart 3proxy
docker compose restart telemt

# Обновить сервисы
docker compose pull && docker compose up -d
```

## Функции Безопасности

- **Аутентификация**: Все прокси-сервисы требуют логин/пароль
- **TLS Шифрование**: Telegram прокси использует маскировку FakeTLS
- **SSL Сертификаты**: Автоматические сертификаты Let's Encrypt с обновлением
- **Лимиты Ресурсов**: Ограничения контейнеров предотвращают злоупотребление ресурсами
- **Изоляция Сети**: Сервисы используют отдельные Docker сети

### Конфигурация Fail2Ban

Скрипт развертывания автоматически устанавливает и настраивает Fail2Ban со следующей защитой:

- **SSH Защита**: Блокирует попытки подбора SSH пароля (3 попытки, бан на 1 час)
- **HTTP Аутентификация**: Блокирует неудачные попытки аутентификации прокси
- **Ограничение Скорости**: Предотвращает чрезмерные запросы к веб-сервисам
- **Защита от Ботов**: Блокирует вредоносный бот-трафик

**Команды Управления Fail2Ban:**
```bash
# Проверить статус Fail2Ban
sudo fail2ban-client status

# Проверить статус конкретной jail
sudo fail2ban-client status sshd
sudo fail2ban-client status nginx-http-auth

# Разблокировать IP
sudo fail2ban-client set sshd unbanip IP_АДРЕС

# Просмотреть заблокированные IP
sudo fail2ban-client banned
```

## Устранение Неполадок

### Конфликты Портов
- Убедитесь, что порты 80, 443, 8080 не заняты
- Проверьте настройки файрвола: `sudo ufw status`

### Проблемы с Docker
- Проверьте, что Docker запущен: `sudo systemctl status docker`
- Проверьте версию Docker Compose: `docker compose version`

### Проблемы с Сертификатами
- Убедитесь, что A-запись домена указывает на IP VPS
- Проверьте логи certbot: `docker compose logs certbot`

### Проблемы с Производительностью
- Мониторинг использования ресурсов: `docker stats`
- Проверьте логи контейнеров на наличие ошибок
